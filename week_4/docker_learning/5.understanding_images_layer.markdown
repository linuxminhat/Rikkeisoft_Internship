Kiến trúc dựa trên lớp của Docker : 
-Docker dùng kiến trúc dựa trên lớp xây dựng và quản lí Docker Image. 
-Kiến trúc này giúp Docker tạo ra các image nhẹ hơn và hiệu quả dễ dàng chia sẻ hơn
-----
giải thích : Docker image không phải đơn lẻ mà nó là sự kết hợp của nhiều lớp (layer) được xếp chồng lên nhau 
-----
Các lệnh trong Dockerfile tạo ra lớp mới : Ví dụ như FROM,RUN,COPY và ADD 
-----
Docker lưu trữ nhiều bản sao của một image nếu chúng dùng chung 1 lớp, giúp tiết kiệm thời gian và không gian hệ thống 
VD : từ kinh nghiệm của bản thân, nhận ra thời gian chạy một Dockerfile lúc ban đầu lúc nào cũng lâu hơn so với lúc sau. Nếu lúc sau chỉ cần hiệu chỉnh 1 tí trên project ban đầu.
-----
Ưu điểm : 
- Tối ưu hóa không gian : lưu trữ lớp đầu tiên và chia sẻ nhiều image 
- Sử dụng cache hiệu quả
- Quản lí phiên bản
-----
Ví dụ về kiến trúc lớp với Dockerfile
Hãy xem xét một ví dụ đơn giản về Dockerfile:

dockerfile
Copy code
FROM node:14
WORKDIR /app
COPY package.json .
RUN npm install
COPY . .
CMD ["node", "app.js"]
Các lớp sẽ được tạo ra như sau:

Lớp 1: FROM node:14 - Lớp nền từ image Node.js phiên bản 14.
Lớp 2: WORKDIR /app - Tạo thư mục làm việc /app.
Lớp 3: COPY package.json . - Sao chép file package.json.
Lớp 4: RUN npm install - Cài đặt các thư viện từ package.json.
Lớp 5: COPY . . - Sao chép toàn bộ mã nguồn vào image.
Lớp 6: CMD ["node", "app.js"] - Lệnh khởi động container.
