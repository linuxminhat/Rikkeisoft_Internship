cgroups, viết tắt của control groups, là tính năng của kernel linux cho phép quản lí và phân bổ tài nguyên hệ thống
VD : CPU, bộ nhớ, băng thông mạng cho các nhóm tiến trình khác nhau chạy trong hệ thống
----------
Vai trò cgroups trong Docker : 
1/giới hạn tài nguyên : Thiết lập giới hạn tài nguyên mà container có thể dùng 
VD : CPU tối đa, bộ nhớ RAM hoặc băng thông mạng cho phép 
2/cách ly tài nguyên : đảm bảo các tài nguyên phân bổ cho container cụ thể không 
bị ảnh hưởng bởi các container khác hoặc hệ điều hành chính 
3/giám sát tài nguyên : giám sát thông qua cgroups 
4/quản lý ưu tiên : đặt các container vào ưu tiên khác nha u
----------
LƯU Ý : Các lệnh liên quan tới cgroups không phải là một phần trực tiếp của Docker, 
mà thay vào đó nó chỉ là một tuỳ chọn để quản lí tài nguyên
VD : 
1/Giới hạn CPU 
--cpus=<value>: Giới hạn số lượng CPU mà container có thể sử dụng
docker run --cpus="1.5" my_container
--cpu-shares=<value>: Đặt mức ưu tiên CPU cho container so với các container khác. Giá trị mặc định là 1024.
docker run --cpu-shares=512 my_container
--cpuset-cpus=<range>: Chỉ định các CPU cụ thể mà container có thể sử dụng.
bash
docker run --cpuset-cpus="0,1" my_container
2/Giới hạn bộ nhớ : 
-m hoặc --memory=<value>: Giới hạn lượng bộ nhớ RAM mà container có thể sử dụng.
bash
docker run -m 512m my_container
--memory-swap=<value>: Giới hạn tổng bộ nhớ và swap mà container có thể sử dụng.
docker run -m 512m --memory-swap=1g my_container
